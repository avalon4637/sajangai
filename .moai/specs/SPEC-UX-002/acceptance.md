# SPEC-UX-002 Acceptance Criteria

<!-- TAG: SPEC-UX-002 -->

## Criteria

1. [ ] AI 에이전트 응답의 모든 시간이 KST로 표시된다
2. [ ] 24시간 이내 시간은 상대 시간("3시간 전")으로 표시된다
3. [ ] 사이드바 메뉴가 논리적 그룹으로 구분되어 아코디언으로 표시된다
4. [ ] 생존 점수 하단에 맥락 설명이 표시된다
5. [ ] 생존 점수 옆에 "이 점수를 올리려면?" CTA가 표시된다
6. [ ] 고정비 페이지에 지출 연동 안내 문구가 표시된다

## Test Cases

### TC-01: AI 응답 시간대 KST 표시

```
Given: AI 에이전트가 2026-04-05T05:30:00Z (UTC)에 응답을 생성했다
When:  채팅 화면에서 해당 메시지를 확인한다
Then:  "2026.04.05 14:30 (KST)" 또는 "3시간 전" 형식으로 표시된다
And:   UTC 시간이 직접 노출되지 않는다
```

### TC-02: 상대 시간 표시

```
Given: AI 응답이 현재로부터 2시간 전에 생성되었다
When:  채팅 화면에서 해당 메시지를 확인한다
Then:  "2시간 전"으로 표시된다
```

### TC-03: 사이드바 아코디언

```
Given: 대시보드 페이지에 접근한다
When:  사이드바를 확인한다
Then:  메뉴가 "매장 관리", "AI 팀", "분석" 등의 그룹으로 나뉘어 표시된다
And:   그룹을 클릭하면 하위 메뉴가 접히거나 펼쳐진다
And:   현재 활성 페이지의 그룹은 자동으로 펼쳐져 있다
```

### TC-04: 생존 점수 맥락

```
Given: 생존 점수가 45점이다
When:  생존 점수 게이지를 확인한다
Then:  점수 하단에 "위험 구간입니다. 지출을 점검해 주세요" 텍스트가 표시된다
And:   "이 점수를 올리려면?" CTA 버튼이 표시된다
```

### TC-05: 고정비 안내 문구

```
Given: 고정비 관리 페이지에 접근한다
When:  페이지 상단을 확인한다
Then:  "고정비는 매월 자동으로 지출에 반영됩니다" 안내 문구가 Alert로 표시된다
```

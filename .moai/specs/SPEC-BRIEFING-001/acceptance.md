# SPEC-BRIEFING-001 Acceptance Criteria

<!-- TAG: SPEC-BRIEFING-001 -->

## Criteria

1. [ ] 대시보드 진입 시 "오늘의 브리핑" 카드가 최상단에 표시된다
2. [ ] 브리핑 카드에 매출 요약(어제 매출, 전일/전주 대비 증감률)이 표시된다
3. [ ] 브리핑 카드에 미답변 리뷰 수와 부정 리뷰 요약이 표시된다
4. [ ] 이상 징후 감지 시 빨간색 뱃지와 경고 아이콘이 표시된다
5. [ ] 데이터 로딩 중 스켈레톤 UI가 표시된다
6. [ ] 신규 사용자(매출 데이터 없음)에게 온보딩 메시지가 표시된다
7. [ ] Server Component에서 데이터를 fetch한다
8. [ ] 모바일/데스크톱 반응형 레이아웃이 정상 동작한다

## Test Cases

### TC-01: 브리핑 카드 정상 표시 (데이터 있는 사용자)

```
Given: 사용자가 revenues 테이블에 최근 7일 매출 데이터를 보유하고 있다
When:  대시보드 페이지에 진입한다
Then:  "오늘의 브리핑" 카드가 페이지 최상단에 표시된다
And:   어제 총 매출 금액이 원화 형식으로 표시된다
And:   전일 대비 증감률이 표시된다 (양수: 초록색, 음수: 빨간색)
And:   전주 동일 요일 대비 증감률이 표시된다
```

### TC-02: 리뷰 알림 표시

```
Given: 사용자의 매장에 미답변 리뷰 3건이 존재한다
And:   그 중 부정 리뷰(sentiment_score < 0.3)가 1건 존재한다
When:  대시보드 페이지에 진입한다
Then:  "미답변 리뷰 3건" 텍스트와 주황색 뱃지가 표시된다
And:   부정 리뷰의 요약 텍스트가 표시된다
```

### TC-03: 이상 징후 감지 및 표시

```
Given: 사용자의 어제 매출이 최근 7일 평균 대비 35% 감소했다
When:  대시보드 페이지에 진입한다
Then:  이상 징후 섹션에 "매출이 평소 대비 35% 감소" 메시지가 표시된다
And:   빨간색 뱃지와 AlertTriangle 아이콘이 표시된다
```

### TC-04: 이상 징후 없음

```
Given: 사용자의 매출/지출 데이터가 정상 범위 내이다
When:  대시보드 페이지에 진입한다
Then:  이상 징후 섹션에 "특이사항 없어요" 또는 유사한 안심 메시지가 표시된다
And:   경고 아이콘이 표시되지 않는다
```

### TC-05: 신규 사용자 온보딩

```
Given: 사용자가 사업장을 등록했으나 매출 데이터가 없다
When:  대시보드 페이지에 진입한다
Then:  "데이터를 연결하면 AI 점장이 매일 브리핑을 준비합니다" 메시지가 표시된다
And:   "배달앱 연결하기", "수동으로 매출 입력하기" CTA가 표시된다
And:   일반 브리핑 카드는 표시되지 않는다
```

### TC-06: 스켈레톤 로딩

```
Given: 대시보드 페이지에 진입하여 브리핑 데이터 로딩 중이다
When:  Server Component가 데이터를 fetch하는 동안
Then:  브리핑 카드 영역에 스켈레톤 UI가 표시된다
And:   스켈레톤은 실제 카드와 동일한 크기/레이아웃을 가진다
```

### TC-07: 매출 증감률 색상 코딩

```
Given: 어제 매출이 전일 대비 12.3% 증가했다
When:  브리핑 카드가 렌더링된다
Then:  "+12.3%" 텍스트가 초록색으로 표시된다

Given: 어제 매출이 전주 대비 3.1% 감소했다
When:  브리핑 카드가 렌더링된다
Then:  "-3.1%" 텍스트가 빨간색으로 표시된다
```

### TC-08: 모바일 반응형

```
Given: 모바일 뷰포트(375px)에서 대시보드에 접근한다
When:  브리핑 카드가 렌더링된다
Then:  매출/리뷰/이상징후 섹션이 세로 스택으로 배치된다
And:   텍스트가 잘리지 않고 모두 표시된다
```

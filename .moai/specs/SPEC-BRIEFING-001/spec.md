## Problem Statement

현재 대시보드 첫 화면이 너무 비어 있다. 사용자가 로그인 직후 "이 서비스가 나를 위해 무엇을 하고 있는지" 즉시 인지할 수 없다. 소상공인 사용자는 바쁜 영업 중에 앱을 열기 때문에, 첫 화면에서 **오늘의 핵심 정보**를 3초 이내에 전달하지 못하면 서비스 가치를 체감하지 못한다.

이것은 PMF의 핵심 -- **"수동에서 능동으로"** 전환의 첫 번째 접점이다. AI 점장이 알아서 챙겨주는 경험은 대시보드를 열었을 때 시작되어야 한다.

### 현재 상태

- 대시보드에 KPI 카드는 있으나 정적 숫자만 표시
- 사용자가 직접 각 메뉴를 탐색해야 인사이트를 얻을 수 있음
- "오늘 뭐가 달라졌는지" 한눈에 파악 불가

### 목표 상태

- 로그인 즉시 "오늘의 브리핑" 카드가 대시보드 상단에 표시
- AI 점장이 매출/리뷰/이상징후를 요약하여 자연어로 전달
- 사용자는 아무 것도 클릭하지 않아도 핵심 정보를 얻음

## Requirements

### Must Have

1. **WHEN** 사용자가 대시보드에 진입할 **THEN** "오늘의 브리핑" 카드가 대시보드 최상단에 표시되어야 한다
2. **WHEN** 오늘의 브리핑 카드가 렌더링될 **THEN** 다음 3개 영역이 반드시 포함되어야 한다:
   - **매출 요약**: 어제 총 매출, 전일/전주 동일 대비 증감률
   - **리뷰 알림**: 미답변 리뷰 수, 최근 부정 리뷰 요약
   - **이상 징후**: 매출 급감/급증, 비정상 지출, 미정산 건 등
3. **WHEN** 브리핑 데이터가 로딩 중일 **THEN** 스켈레톤 UI가 표시되어야 한다
4. **IF** 매출 데이터가 아직 없는 신규 사용자일 **THEN** "데이터를 연결하면 AI 점장이 매일 브리핑을 준비합니다" 온보딩 메시지를 표시해야 한다
5. **WHEN** 이상 징후가 감지될 **THEN** 해당 항목에 시각적 강조(빨간색 뱃지, 아이콘)가 적용되어야 한다
6. 시스템은 **항상** 브리핑 카드의 데이터를 Server Component에서 fetching하고, 캐싱하여 빠른 로딩을 보장해야 한다

### Nice to Have

7. 브리핑 카드에 "자세히 보기" 링크로 해당 상세 페이지(매출 분석, 리뷰 관리 등)로 이동
8. 브리핑 카드 하단에 "점장에게 물어보기" CTA로 AI 채팅 진입
9. 시간대별 인사말 변경 (아침: "좋은 아침이에요", 오후: "오늘 하루도 수고 많으셨어요")
10. 브리핑 카드 접기/펼치기 토글 (접은 상태를 localStorage에 저장)

## UI Design Notes

### 브리핑 카드 레이아웃

```
+----------------------------------------------------------+
|  [점장 아바타]  오늘의 브리핑            2026.04.05 (토)   |
+----------------------------------------------------------+
|                                                          |
|  [매출 아이콘]  어제 매출 1,247,000원                     |
|                 전일 대비 +12.3% | 전주 대비 -3.1%        |
|                                                          |
|  [리뷰 아이콘]  미답변 리뷰 3건                           |
|                 "음식이 식어서 왔어요" 외 1건 부정 리뷰     |
|                                                          |
|  [경고 아이콘]  이상 징후 1건                              |
|                 화요일 매출이 평소 대비 35% 감소            |
|                                                          |
+----------------------------------------------------------+
|  [자세히 보기]              [점장에게 물어보기 ->]          |
+----------------------------------------------------------+
```

### 카드 디자인 원칙

- **배경**: 흰색 카드 + 미세한 그림자 (`shadow-sm`)
- **점장 아바타**: 32x32 원형, 브랜드 컬러 배경
- **매출 증감**: 양수=초록색, 음수=빨간색
- **이상 징후**: 빨간색 뱃지 + 경고 아이콘 (AlertTriangle)
- **리뷰 알림**: 미답변 건수 강조 (주황색 뱃지)
- **반응형**: 모바일에서는 세로 스택, 데스크톱에서는 가로 3컬럼도 고려

### 데이터 소스 매핑

| 브리핑 영역 | 데이터 소스 | 테이블 |
| ----------- | ---------- | ------ |
| 어제 매출 | revenues 테이블 SUM(amount) WHERE date = yesterday | `revenues` |
| 전일 대비 | (어제 - 전일) / 전일 * 100 | `revenues` |
| 전주 대비 | (어제 - 7일전 동일요일) / 7일전 * 100 | `revenues` |
| 미답변 리뷰 | reviews WHERE replied_at IS NULL | `reviews` (agent DB) |
| 부정 리뷰 | reviews WHERE sentiment_score < 0.3 | `reviews` (agent DB) |
| 이상 징후 - 매출 | 최근 7일 평균 대비 30% 이상 감소 | `revenues` |
| 이상 징후 - 지출 | 전월 동기 대비 50% 이상 증가 | `expenses` |
| 이상 징후 - 미정산 | 카드 매출 중 정산 미완료 건 | `api_connections` (향후) |

## Technical Notes

### 구현 방향

- **Server Component 기반**: 대시보드 페이지에서 Server Component로 브리핑 데이터 fetch
- **병렬 데이터 로딩**: `Promise.all`로 매출/리뷰/이상징후 데이터를 병렬 fetch
- **Suspense + Skeleton**: 각 영역을 `<Suspense>` 경계로 나누어 개별 로딩 가능
- **Server Action 캐싱**: `unstable_cache` 또는 `revalidatePath`로 데이터 캐싱 (1시간 TTL)

### 이상 징후 감지 로직

```
매출 급감: 어제 매출 < (최근 7일 평균 * 0.7)
매출 급증: 어제 매출 > (최근 7일 평균 * 1.5)
지출 이상: 이번 달 지출 > (전월 동기 * 1.5)
미정산:    카드매출 중 정산예정일 초과 건 (향후 구현)
```

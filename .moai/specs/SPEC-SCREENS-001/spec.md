## Environment

- **Runtime**: Next.js 16 App Router, React 19, TypeScript 5.9
- **UI**: shadcn/ui (new-york style), Tailwind CSS 4, Recharts
- **Database**: Supabase PostgreSQL 16 with Row-Level Security
- **Data Sources**: daily_reports, monthly_summaries, delivery_reviews, store_context, Hyphen revenue data
- **Design Reference**: pencil-new.pen for mobile responsive layouts

## Assumptions

- A1: SPEC-JEONGJANG-001 daily_reports table is populated with briefings
- A2: SPEC-SERI-001 produces revenue analysis available in store_context
- A3: SPEC-DAPJANGI-001 delivery_reviews has sentiment_score and keywords populated
- A4: monthly_summaries table has aggregated KPI data
- A5: Hyphen API provides channel-level revenue breakdowns
- A6: All pages must load within 2 seconds
- A7: Mobile responsive design follows pencil-new.pen specifications

## Requirements

### S1: Dashboard (/dashboard) - Jeongjang Home

**[REQ-S1-01]** WHEN a user navigates to /dashboard THEN the system shall display the latest daily briefing from daily_reports.

**[REQ-S1-02]** The system shall always display real-time KPI cards showing total revenue, total expense, net profit, and survival score from monthly_summaries and live revenue data.

**[REQ-S1-03]** WHEN action items exist from Jeongjang proactive diagnosis THEN the system shall display them as a prioritized list.

**[REQ-S1-04]** The system shall always show agent status indicators displaying the last run time for Seri, Dapjangi, and Viral agents.

**[REQ-S1-05]** The system shall always render a chat input at the bottom of the dashboard for Jeongjang conversation.

### S2: Analysis (/analysis) - Seri Revenue View

**[REQ-S2-01]** WHEN a user navigates to /analysis THEN the system shall display a calendar heatmap populated from actual revenue records.

**[REQ-S2-02]** WHEN Hyphen delivery and card data is available THEN the system shall show a channel breakdown chart (Baemin, Coupang Eats, Yogiyo, card sales).

**[REQ-S2-03]** WHEN historical data spans 4+ weeks THEN the system shall display day-of-week revenue patterns.

**[REQ-S2-04]** Where possible, the system shall display an AI insight card generated by the Seri agent.

### S3: Review (/review) - Dapjangi Review View

**[REQ-S3-01]** WHEN a user navigates to /review THEN the system shall display a review list from the delivery_reviews table with real data.

**[REQ-S3-02]** WHEN reviews have been analyzed THEN the system shall display sentiment analysis results including keywords and scores.

**[REQ-S3-03]** WHEN replies with status "pending" exist THEN the system shall display an auto-reply approval queue.

**[REQ-S3-04]** The system shall always show reply status tracking for each review (auto, pending, published, skipped).

### S4: Empty States

**[REQ-S4-01]** IF no data exists for a page THEN the system shall display a guided empty state directing the user to connect data sources in settings.

**[REQ-S4-02]** The system shall not display broken charts or "undefined" values when data is missing.

### Data Flow

```
/dashboard
  |-- Server Component: fetch daily_reports (latest), monthly_summaries, store_context
  |-- Client Components: KPI cards, briefing card, action items, agent status, chat input

/analysis
  |-- Server Component: fetch revenue records, Hyphen channel data
  |-- Client Components: calendar heatmap, channel breakdown, day-of-week patterns, AI insight

/review
  |-- Server Component: fetch delivery_reviews with sentiment data
  |-- Client Components: review list, reply queue, sentiment chart, status tracking
```

### Constraints

- C1: All pages must load within 2 seconds (use Server Components for data fetching)
- C2: Mobile responsive following pencil-new.pen design specifications
- C3: Empty states must be user-friendly with clear CTAs to settings
- C4: No mock data in production builds; all data from real database tables
- C5: Charts must handle zero-data gracefully (show empty state, not errors)
- C6: All queries scoped to authenticated business_id via RLS

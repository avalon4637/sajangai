# SPEC-UI-002 Acceptance Criteria

<!-- TAG: SPEC-UI-002 -->

## Criteria

1. [ ] 대시보드 KPI 카드에 데이터 기간이 명시되어 있다
2. [ ] 거래처 카드에서 편집/삭제/상세 접근이 가능하다
3. [ ] 거래처 삭제 시 확인 다이얼로그가 표시된다
4. [ ] 계산서 매입 탭에 데이터 없을 때 empty state가 표시된다
5. [ ] empty state에 데이터 추가 안내 CTA가 포함되어 있다
6. [ ] 고정비 카테고리에 "대출 상환" 옵션이 존재한다

## Test Cases

### TC-01: KPI 카드 기간 표시

```
Given: 대시보드 페이지에 접근한다
When:  상단 KPI 카드를 확인한다
Then:  각 카드에 "이번 달", "최근 7일" 등 기간 라벨이 표시된다
And:   라벨이 실제 데이터 조회 범위와 일치한다
```

### TC-02: 거래처 편집

```
Given: 거래처 목록에 "OO식자재" 카드가 표시된다
When:  카드의 점 3개 메뉴를 클릭하고 "편집"을 선택한다
Then:  거래처 정보 수정 폼이 표시된다
And:   이름, 연락처 등 필드를 수정할 수 있다
And:   저장 시 변경 내용이 반영된다
```

### TC-03: 거래처 삭제

```
Given: 거래처 카드의 점 3개 메뉴에서 "삭제"를 선택한다
When:  확인 다이얼로그가 표시된다
And:   "삭제" 버튼을 클릭한다
Then:  해당 거래처가 목록에서 제거된다
And:   성공 토스트 메시지가 표시된다
```

### TC-04: 계산서 매입 empty state

```
Given: 매입 계산서 데이터가 0건이다
When:  계산서 페이지의 "매입" 탭을 선택한다
Then:  "아직 매입 계산서가 없습니다" 메시지가 표시된다
And:   데이터 추가 방법을 안내하는 CTA 버튼이 표시된다
```

### TC-05: 고정비 대출 상환 카테고리

```
Given: 고정비 등록 폼을 연다
When:  카테고리 드롭다운을 확인한다
Then:  "대출 상환" 카테고리가 옵션에 존재한다
```

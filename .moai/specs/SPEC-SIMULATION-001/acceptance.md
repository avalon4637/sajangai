# SPEC-SIMULATION-001 수락 기준: What-if 시뮬레이션 UI

---

## AC-01: 시나리오 선택

```gherkin
Scenario: 4가지 시나리오 표시
  Given /dashboard/simulation 페이지에 접속한다
  Then 4가지 시나리오 카드가 표시된다:
    | 시나리오 | 설명 |
    | 직원 변동 | "직원을 고용/감원하면?" |
    | 매출 변동 | "매출이 변하면?" |
    | 임대료 변동 | "임대료가 변하면?" |
    | 매입 변동 | "재료비가 변하면?" |
```

## AC-02: 시뮬레이션 실행

```gherkin
Scenario: 직원 추가 시뮬레이션
  Given 현재 인건비가 200만원이다
  And "직원 변동" 시나리오를 선택했다
  When 값에 "2000000"(원)을 입력하고 "시뮬레이션 실행"을 클릭한다
  Then 변경 전/후 KPI가 나란히 표시된다
  And 인건비 비율과 고정비 비율이 증가한 것이 빨간색으로 표시된다
  And 생존점수 변화가 시각적으로 표시된다

Scenario: 프리셋 사용
  Given "직원 변동" 시나리오를 선택했다
  When "알바 1명 추가 (월 200만원)" 프리셋을 클릭한다
  Then 값이 2000000으로 자동 입력된다

Scenario: 비율 입력
  Given "매출 변동" 시나리오를 선택했다
  When 입력 모드를 "%"로 전환하고 "10"을 입력한다
  Then 현재 매출의 10% 증가가 시뮬레이션된다
```

## AC-03: 결과 비교

```gherkin
Scenario: KPI 비교 표시
  Given 시뮬레이션이 실행되었다
  Then 다음 항목의 변경 전/후가 표시된다:
    | 항목 | 변경 전 | 변경 후 | 변화 |
    | 순이익 | 기존값 | 계산값 | 차이 + % |
    | 생존점수 | 기존값 | 계산값 | 차이 |
    | 매출총이익률 | 기존값 | 계산값 | 차이 |
    | 인건비 비율 | 기존값 | 계산값 | 차이 |
    | 고정비 비율 | 기존값 | 계산값 | 차이 |

Scenario: 데이터 없는 경우
  Given 현재 월 KPI 데이터가 없다
  When 시뮬레이션 페이지에 접속한다
  Then "먼저 매출/비용 데이터를 입력해주세요" 메시지가 표시된다
```

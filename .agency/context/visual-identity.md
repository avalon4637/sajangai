## Typography

### Heading Font

Family: -apple-system, BlinkMacSystemFont, "Segoe UI", "Noto Sans KR", sans-serif

Weights: 700 (h1 KPI numbers), 600 (h2-h3), 500 (h4-h6)

### Body Font

Family: -apple-system, BlinkMacSystemFont, "Segoe UI", "Noto Sans KR", sans-serif

Weight: 400 (regular)

Line Height: 1.6

### Monospace Font

Family: "JetBrains Mono", "Fira Code", monospace

### Size Scale

Base: 14px (dashboard context — denser than marketing pages)

Scale:
- xs: 11px (badges, tags)
- sm: 13px (captions, secondary info)
- base: 14px (body)
- md: 15px (emphasized body)
- lg: 18px (sub-headings)
- xl: 22px (section titles)
- 2xl: 28px (KPI values)
- 3xl: 36px (ROI KPI — key metric emphasis)
- 4xl: 48px (hero only)

## Logo Usage

Primary Logo: "사장 AI" wordmark — system font, semibold, #18181B with #4B6BF5 accent

Minimum Size: 80px wide

Clear Space: 8px on all sides

Dark Background Variant: white wordmark on #1E293B

Favicon: stylized "사" character or crown/店 icon, 32x32

## Imagery Style

Photography: authentic candid shots of Korean small business owners (restaurant, café, retail)
Avoid: generic stock photos, western food/contexts

Illustrations: flat, minimal, uses brand colors; emoji-style agent avatars acceptable

Icons: Lucide icon set, 20px default, 1.5px stroke
Agent avatars: circular badge with emoji on colored background

Decorative Elements: subtle gradient backgrounds for hero/briefing cards; no heavy patterns

## Layout Preferences

Grid System: Sidebar (240px) + main content, 12-column grid within main
Max content width: 1200px within main area

Spacing Scale: 4px base unit (4, 8, 12, 16, 20, 24, 32, 40, 48, 64)

Border Radius:
- sm: 6px (tags, small buttons)
- md: 8px (buttons, inputs)
- lg: 12px (cards)
- xl: 16px (briefing card, large containers)
- full: 9999px (badges, avatars)

Content Density: medium-dense (data dashboard — needs to show multiple KPIs)
Card Style: 1px border (#E4E4E7), shadow-sm, radius-lg
Sidebar: fixed left, 240px, bg #F8F9FA

## Design Anti-Patterns

- No generic purple/blue gradient hero (the "AI slop" look)
- No heavy drop shadows (max shadow-md)
- No centered body text
- No more than 3 font weights on a page
- No rainbow multi-color gradients
- No oversized empty whitespace on data dashboard pages (density matters)
- No decorative illustrations that compete with data
- No English-only UI text (all UI copy must be Korean)
- Do not use "구독" framing — always "에이전트 월급" or "점장 고용"

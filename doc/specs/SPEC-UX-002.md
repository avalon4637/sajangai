# SPEC-UX-002: Mobile-First Agent UX Redesign

## Goal
각 에이전트 페이지를 모바일 최우선으로 재설계하여 "폰에서 30초 확인" 경험 달성.
핵심: 사장님이 매장에서 바쁜 중에 폰으로 빠르게 확인하고 행동하는 시나리오.

---

## A. 세리 — 매출 분석 모바일 최적화

### 현재 문제
- KPI 카드(생존 게이지 + P&L 3열)가 화면 전체를 차지
- 가장 중요한 **달력**이 스크롤 최하단으로 밀림
- 모바일에서 스크롤 3-4회 해야 달력 도달

### 개선 방향

#### A-1. 모바일 컴팩트 KPI 스트립
모바일에서 생존 게이지 + P&L 카드를 **한 줄 스트립**으로 압축:
```
[11점 F등급] 매출 382만 | 순이익 +107만 | 현금 325만
```
- 생존 점수: 원형 게이지 → 점수+등급 배지만 (탭하면 상세 열림)
- P&L 4카드 → 핵심 3개 숫자 한 줄 (매출/순이익/현금)
- 데스크톱: 기존 레이아웃 유지

#### A-2. 달력 최우선 배치
모바일 렌더링 순서 변경:
```
Before: 타이틀 → 게이지 → KPI카드 → 달력 → AI분석
After:  타이틀 → 컴팩트KPI → 달력 → AI분석(접기)
```

#### A-3. AI 분석 접기
- "세리의 분석" 사이드패널 → 모바일에서 접기 가능한 Accordion
- 기본: 접힌 상태 (달력이 바로 보이도록)

- `src/app/(dashboard)/analysis/page-client.tsx`
- `src/components/seri/pnl-summary-cards.tsx`
- `src/components/seri/survival-gauge.tsx`

## B. 답장이 — 리뷰 관리 모바일 최적화

### 현재 문제
- KPI 카드 4개가 크게 표시됨
- 기본 필터가 "전체" → 이미 답변한 리뷰도 섞여서 노이즈
- 복사하기 + 딥링크 이동이 메인 작업인데 접근 어려움
- 답글 달면 같은 목록에 계속 남아있음

### 개선 방향

#### B-1. 모바일 컴팩트 KPI
4개 카드 → 한 줄 요약:
```
리뷰 4건 | ★4.0 | 미답변 3건
```
- 핵심 3개만: 총 리뷰, 평균 별점, 미답변 수
- 감성 점수/자동답글률은 모바일에서 숨김 (데스크톱만)

#### B-2. 기본 필터 = 미답변
- `activeFilter` 기본값: `"all"` → `"unanswered"`
- 첫 화면에 미답변 리뷰만 표시
- 답글 완료 시 자동으로 목록에서 사라짐 (optimistic update)

#### B-3. 리뷰 액션 버튼 강화
각 리뷰 카드에 바로 보이는 액션:
```
┌──────────────────────────┐
│ ★★★★★ 배민 | 골프초보    │
│ 시설이 깨끗하고 직원분이...│
│                          │
│ AI 답글: "감사합니다..."  │
│ [📋 복사] [🔗 답글 쓰러가기] [✓ 완료] │
└──────────────────────────┘
```
- **복사**: 클립보드 복사 + toast "복사되었습니다"
- **답글 쓰러가기**: 배민/쿠팡/요기요/네이버 딥링크
- **완료**: 상태를 "발행완료"로 변경, 목록에서 제거

#### B-4. 모바일 리뷰 목록 전용 레이아웃
- 데스크톱: 좌측 리스트 + 우측 상세 패널 (현재 유지)
- 모바일: **단일 리스트** (상세 패널 제거)
  - 각 리뷰 카드에 AI 답글 + 액션 버튼 인라인 표시
  - 탭하면 확장 (Accordion 스타일)

## C. 점장 — 홈 브리핑 + 컨텐츠 시스템

### 현재 문제
- KPI가 너무 간소 (매출 1줄 + 추세 배지만)
- 채팅창에 환영 메시지 1개만 있고 빈 공간이 대부분
- 카카오톡 vs 앱 내 컨텐츠가 정리되지 않음
- 마지막 보고 컨텐츠가 채팅에 안 보임

### C-1. 컨텐츠 종류 정리

#### 카카오톡 발송 컨텐츠 (간단 텍스트 + 딥링크)
| 종류 | 내용 | 딥링크 |
|------|------|--------|
| 아침 브리핑 | "어제 매출 48만원(+12%) / 리뷰 2건" | /dashboard |
| 긴급 리뷰 | "부정 리뷰 1건 접수 — 배달 느림 불만" | /review |
| 자금 경고 | "3일 후 자금 부족 예상 — 현재 잔고 120만" | /analysis |
| 이상 감지 | "매출 -32% 감지 — 원인 분석 완료" | /dashboard |
| 주간 요약 | "이번 주 매출 280만 / 지난 주 대비 +8%" | /analysis |
| 월간 ROI | "점장 성과: 절감 47만 / 수익 28만" | /billing |

#### 앱 내 컨텐츠 (시각화 + 인터랙션)
| 종류 | 시각화 | 위치 |
|------|--------|------|
| 일일 브리핑 카드 | KPI 그리드 + 추세 차트 + 벤치마크 | 점장 홈 상단 |
| 인사이트 카드 | severity 아이콘 + 제목 + 요약 + 액션 버튼 | 점장 홈 피드 |
| 리뷰 알림 카드 | 별점 + 플랫폼 + 본문 미리보기 + 답글 CTA | 점장 홈 피드 |
| 자금 경고 카드 | 잔고 게이지 + 위험일 표시 + 시뮬레이션 링크 | 점장 홈 피드 |
| 주간 리포트 | 7일 매출 차트 + 요약 텍스트 | 점장 홈 피드 |

### C-2. 점장 홈 레이아웃 재설계

#### 모바일 레이아웃:
```
┌─────────────────────────────────────┐
│ [KPI 스트립] 어제 92만▼ | 이달 382만 │ ← 항상 고정
├─────────────────────────────────────┤
│ ┌─ 최근 브리핑 카드 ──────────────┐ │
│ │ ☀️ 아침 리포트 (오전 8:00)      │ │
│ │ 어제 매출 487,000원 (+12%)     │ │
│ │ ┌──────┬──────┬──────┐         │ │
│ │ │매출   │순이익 │수익률 │         │ │
│ │ │382만  │107만  │28%   │         │ │
│ │ └──────┴──────┴──────┘         │ │
│ │ 📊 주간추세: ▲8% | 🎯 목표 76% │ │
│ └────────────────────────────────┘ │
│                                     │
│ ┌─ 인사이트 피드 ────────────────┐ │
│ │ 🔴 매출 -32% 감지              │ │
│ │ 리뷰 미답변 3건이 원인 추정     │ │
│ │ [답글 확인] [상세 보기]         │ │
│ └────────────────────────────────┘ │
│                                     │
│ ┌─ 리뷰 알림 ────────────────────┐ │
│ │ ⭐ ★★☆☆☆ 배민 부정 리뷰       │ │
│ │ "배달이 너무 늦었어요..."       │ │
│ │ [AI 답글 보기] [답글 쓰러가기]  │ │
│ └────────────────────────────────┘ │
│                                     │
│ ─── 점장에게 물어보기 ───          │
│ [요즘 장사 어때?] [매출 분석해줘]   │
├─────────────────────────────────────┤
│ [채팅 입력] 점장에게 물어보세요... 📤│ ← 항상 하단 고정
└─────────────────────────────────────┘
```

#### 핵심 변경:
1. **KPI 스트립**: 항상 상단 고정, 데이터 충실
2. **브리핑 카드**: 마지막 브리핑을 시각적 카드로 표시 (chat bubble이 아닌 rich card)
3. **인사이트 피드**: 스코어링된 상위 인사이트를 카드로 (채팅 메시지가 아닌 독립 카드)
4. **리뷰 알림**: 미답변 부정 리뷰가 있으면 카드로 표시
5. **추천 질문**: 컨텍스트 기반 동적 추천 (하드코딩 아님)
6. **채팅**: 하단 입력 항상 보임, 질문하면 채팅 모드로 전환

### C-3. 브리핑 Rich Card 컴포넌트
카톡에서 간단 텍스트로 보낸 내용을 앱에서는 시각적으로 표현:

```tsx
// BriefingRichCard — 카톡에서는 텍스트, 앱에서는 시각화
<BriefingRichCard>
  <KpiGrid>  {/* 3열: 매출/순이익/수익률 */}
  <TrendBadge direction="up" value="8%" label="주간" />
  <GoalProgress current={382} target={500} />
  <BenchmarkBadge above={true} diff="+15%" />
</BriefingRichCard>
```

## 실행 계획

### Phase A: 세리 모바일 최적화 (난이도: S)
1. 모바일 컴팩트 KPI 스트립 (sm:hidden 기존 카드)
2. 모바일 렌더링 순서 변경 (달력 우선)
3. AI 분석 패널 접기

### Phase B: 답장이 모바일 최적화 (난이도: M)
1. 모바일 컴팩트 KPI
2. 기본 필터 미답변으로 변경
3. 리뷰 카드 인라인 액션 (복사+딥링크+완료)
4. 모바일 단일 리스트 레이아웃

### Phase C: 점장 홈 재설계 (난이도: L)
1. 컨텐츠 타입 정리 (카톡 vs 앱)
2. 브리핑 Rich Card 컴포넌트
3. 인사이트 피드 카드
4. 리뷰 알림 카드
5. 점장 홈 레이아웃 재구성

### 실행 순서
Phase A → Phase B → Phase C (의존성 순)
A+B 병렬 가능, C는 A,B 이후 (패턴 확립 후 적용)

## 2. 브랜딩 가이드

### 컬러

| 용도 | 컬러 | 비고 |
|------|------|------|
| Primary | #1E40AF (딥블루) | 신뢰, 전문성 |
| Accent | #F59E0B (앰버/골드) | 돈, 성과, 수익 강조 |
| Success | #10B981 (그린) | 상승, 개선, 긍정 |
| Danger | #EF4444 (레드) | 하락, 경고, 위험 |
| BG | #F8FAFC (라이트그레이) | 배경 |
| Text | #1E293B (다크) | 본문 |
| Sub | #64748B (그레이) | 보조 텍스트 |

### 폰트

- 한글: Pretendard (또는 Noto Sans KR)
- 숫자/금액: Pretendard Bold (숫자는 항상 크고 굵게)

### 디자인 원칙

- **숫자를 크게**: 금액, %, 배수는 항상 큰 폰트 + Bold + Accent 컬러
- **카드 UI**: 섹션 내 정보 블록은 둥근 카드로 구성
- **여백 충분히**: 소상공인은 빽빽한 화면 싫어함. 여유 있게
- **아이콘 활용**: 텍스트보다 이모지/아이콘으로 시각적 이해
- **스크롤 애니메이션**: 각 섹션 진입 시 fade-in + slide-up (과하지 않게)

## 3. 페이지 구조 (섹션별 상세)

### 전체 흐름

```
S1. 히어로 — 첫 인상 + 핵심 가치 (3초 안에 이해)
S2. 문제 제기 — "사장님, 이런 거 겪고 계시죠?"
S3. 킥: 체중계 vs 주치의 — 경쟁사 대비 차별화
S4. 핵심 기능 3가지 — 매일 리포트 / 인사이트 / 원클릭 실행
S5. 인사이트 시나리오 쇼케이스 — 실제 카톡 화면 3~4개
S6. ROI 보고서 — "점장이 벌어준 돈"
S7. 에이전트 소개 — 점장/답장이/세리/바이럴
S8. 요금 — 하루 330원
S9. 7일 체험 설계 — D+1~D+7 타임라인
S10. FAQ
S11. 최종 CTA
S12. 푸터
```

**목표**: 3초 안에 "이게 뭔지" 이해시키기

**레이아웃**:
```

│  매일 아침 카톡으로 챙겨드려요.               │
│                                              │
│                                              │
│     ┌──────────────────────────┐             │
│     │  7일 무료 체험 시작하기 →  │             │
│     └──────────────────────────┘             │
```

**카피 가이드**:
- 헤드라인: "하루 330원, 점장 한 명." — 가격+가치를 한 줄에
- 서브: "AI 점장이 매출·리뷰·비용을 매일 아침 카톡으로 챙겨드려요." — 뭘 해주는지 명확히
- CTA: "7일 무료 체험 시작하기" — "구독" 아님, "체험 시작"
- 안심 문구: "카드 등록 없이 바로 시작"

**히어로 이미지**:
- 카톡 아침 리포트 목업 (실제 메시지 UI 스타일)
- 또는: 사장님 손에 폰 들고 있는 일러스트 + 카톡 말풍선

**3가지 핵심 기능**:
1. **매일 아침 카톡 리포트** — 데이터 자동 수집 + 분석 + 카톡 발송
2. **문제 먼저 감지 + 원인 분석** — Proactive 인사이트 (킥)
3. **버튼 하나면 실행** — 답글/문자/프로모션 원클릭

각 기능 카드에 실제 카톡/앱 스크린샷 목업 포함. 목업은 HTML/CSS로 직접 구현 (이미지 아님)

- 각 목업 하단에 [버튼]이 보여야 함 — "버튼 누르면 해결" 임팩트

### S10. FAQ

**목표**: 가입 전 불안 요소 해소

**필수 질문 6개**:

1. **"배민/쿠팡 계정 정보 안전한가요?"**
   → 하이픈(금융보안 인증 업체)이 암호화 저장. sajang.ai는 비밀번호를 보관하지 않아요.

2. **"매일 아침 카톡이 귀찮지 않을까요?"**
   → 사장님이 원하는 시간에, 원하는 스타일로 보내드려요. 설정에서 언제든 조정 가능해요.

3. **"AI가 리뷰 답글을 잘 써주나요?"**
   → 사장님 매장 톤에 맞춰 작성하고, 등록 전에 항상 확인할 수 있어요. 마음에 안 들면 수정도 돼요.

4. **"7일 체험 후 자동 결제되나요?"**
   → 아니요. 체험 끝나면 자동 종료. 사장님이 직접 고용 결정하셔야 해요.

5. **"배달 안 하고 카드매출만 있어도 되나요?"**
   → 네. 카드매출만 연결해도 매출 분석, 비용 경고, 시뮬레이션 모두 사용 가능해요.

6. **"ROI 보고서 숫자, 진짜인가요?"**
   → 보수적 기준으로 계산해요. 과장 없이 실제 데이터 근거로만 산출하고, 계산 방법도 투명하게 공개해요.

**구현**: shadcn Accordion 컴포넌트 사용

## 7. 구현 시 주의사항 (Claude Code용)

### 반드시 지킬 것

1. **모바일 퍼스트**: 모든 레이아웃을 모바일(375px)에서 먼저 디자인, 데스크탑은 확장
2. **카톡 목업은 CSS로**: 이미지 X, HTML/CSS 컴포넌트로 구현 (반응형 + 텍스트 변경 용이)
3. **숫자 강조**: 금액/배수/% 는 항상 text-3xl font-bold + Accent 컬러
4. **CTA 일관성**: 모든 CTA 버튼 텍스트 = "7일 무료 체험 시작하기", Primary 컬러
5. **기존 스택 활용**: Next.js 15 + TailwindCSS + shadcn/ui (새 라이브러리 최소화)
6. **한국어 최적화**: 줄바꿈 word-break: keep-all, 조사 어색함 방지

### 하지 말 것

1. ❌ 영어 섞어 쓰기 (AI Agent → "AI 점장", Proactive → "먼저 알려드려요")
2. ❌ 전문 용어 (ROI → "점장이 벌어준 돈", 크로스 분석 → "원인 추적")
3. ❌ 과장된 표현 ("혁신적", "세계 최초", "획기적")
4. ❌ 복잡한 기능 설명 (AAAS, LangGraph 등 기술 용어 노출)
5. ❌ 너무 많은 텍스트 (각 섹션 카피는 3줄 이내가 이상적)
6. ❌ 스톡 이미지 (실제 UI 목업으로 대체)

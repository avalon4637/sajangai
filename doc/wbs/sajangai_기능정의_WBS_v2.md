## 0. 제품 포지셔닝

| | 캐시노트 | 세일즈랩 | **sajang.ai** |
|---|---------|---------|-------------|
| 본질 | 매출 알림 도구 | 리뷰 답글 도구 | **AI 경영 점장** |
| 하는 일 | "어제 매출 312K" | "리뷰 답글 써드림" | **"매출 떨어진 이유는 이거고, 이렇게 하면 되고, 버튼 누르면 제가 합니다"** |
| 한계 | 숫자만 보여줌 | 단일 작업만 | — |
| 비유 | 체중계 | 물리치료사 | **주치의 (진단+처방+약 타줌+효과 추적)** |

**핵심 공식**:
```
감지 (뭔가 이상함)
  → 원인 분석 (매출×리뷰×비용 크로스 분석)
    → 해결책 (구체적 행동 + 예상 효과 ₩)
      → [버튼 하나] 사장 탭 → AI 실행
        → 결과 추적 (실제 효과 측정)
          → ROI 누적 ("이번 달 점장이 벌어준 돈 75만원")
```

## 1. 기능 체계 총괄

```
F1. 데이터 연동 ─── 하이픈 API로 매출/리뷰/카드 자동 수집
F2. 에이전트 코어 ─ 점장(Supervisor) + 답장이/세리/바이럴
F3. 경영 인사이트 ─ ★ 킥: 25개 시나리오 × 원클릭 실행 × ROI 추적
F4. 개인화 엔진 ── AAAS: 메모리/정책/충돌해결/보상학습
F5. 사장 인터페이스 ─ 카톡 + 웹앱 + 푸시
F6. 비즈니스 인프라 ─ 과금/체험/토큰최적화
```

## 3. F2. 에이전트 코어

### F2.1 에이전트 아키텍처

```
점장 (Supervisor) ─ 전체 조율, 리포트 생성, 사장 대화
  ├── 답장이 (Review) ─ 리뷰 수집/감성분석/AI답글/자동등록
  ├── 세리 (Finance) ─ 매출분석/이상감지/비용분석/시뮬레이션
  └── 바이럴 (Marketing) ─ 이탈감지/재방문문자/프로모션
```

**Tool 설계 (AAAS 원칙: 에이전트당 5개 이내)**:
```
점장: route_to_agent(), generate_report(), send_notification()
답장이: fetch_reviews(), analyze_sentiment(), generate_reply(), post_reply()
세리: query_financial_data(), detect_anomaly(), run_simulation()
바이럴: detect_churn(), generate_message(), send_sms()
```

## 4. F3. 경영 인사이트 엔진 (★ 킥)

> 서비스의 핵심 차별점. "왜 이런 일이 생겼고, 어떻게 하면 되고, 버튼 누르면 제가 합니다"

### F3.1 인사이트 공통 구조

모든 인사이트는 동일한 5단계 구조:

```
[감지] 데이터에서 이상/기회/위험 발견
  → [원인] 크로스 시그널 분석 (매출×리뷰×비용×과거패턴)
    → [해결] 구체적 행동 + 예상 효과(₩)
      → [실행] 버튼 탭 → AI가 대신 수행
        → [추적] 실제 효과 측정 → ROI 누적
```

### F3.3 원클릭 실행 체계

| 실행 액션 | 기술 구현 | 자동화 가능 레벨 |
|----------|----------|----------------|
| 리뷰 답글 등록 | 하이픈 API → 배달앱 게시 | Level 3 (긍정리뷰 자동, 부정은 승인) |
| 재방문 문자 발송 | 알리고/카카오 비즈메시지 API | Level 2 (항상 승인 후 발송) |
| 리뷰 요청 문자 | 최근 5점 고객 자동 선별 → 발송 | Level 3 |
| 감사 문자 자동화 | 첫 주문 다음 날 자동 | Level 4 가능 |
| 프로모션 문구 생성 | AI 생성 → 사장 확인 → 복사 | Level 1 (항상 확인) |
| SNS 리뷰 카드 | AI 문구 + 이미지 생성 | Level 1 |
| 시뮬레이션 실행 | 앱 내 시뮬레이션 화면 이동 | — |
| 가이드 제공 | 단계별 스크린샷 설명 | — |

**실행 플로우 (공통)**:
```
카톡 [버튼] 탭
  → 딥링크 → 앱 실행
  → 미리보기 (수정 가능)
  → [실행] 버튼
  → API 호출 → 완료
  → 총 소요시간: 20~30초
```

### F3.4 ROI 추적 엔진

| 이상 감지 효과 | 과거 유사 하락폭 × 50% | 기여도 절반만 인정 |
| 비용 절감 | 제안 수용 후 실제 절감액 | 사장이 실행한 것만 |
| 단골 복귀 | 문자 후 2주 내 재주문 × 객단가 | 자연 복귀율 10% 제외 |
| 리뷰→평점→매출 | 평점 변화 × 상관계수 × 30% | 보수적 귀속 |
| 시간 절약 | 답글 건수 × 3분 + 리포트 × 10분 | 단순 곱셈 |

### F3.5 ROI 월간 보고서 (카톡)

  절약한 돈    47만원

  아껴준 시간   6시간

  [상세 보기]

## 11. 리스크

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| 하이픈 API 장애 | 높음 | 수집 실패 알림 + 수동 입력 폴백 + SLA 계약 |
| 카톡 알림톡 심사 지연 | 중간 | 앱 푸시 1차, 카톡 2차 |
| 토큰 비용 폭발 | 높음 | Prompt Caching + 요약 + $2/월 목표 |
| 배달앱 답글 등록 제한 | 중간 | 하이픈 지원 여부 사전 확인, 미지원 시 수동 가이드 |
| 과잉 알림 → 이탈 | 높음 | Intervention Timing + 일 3회 상한 + severity 분류 |
| ROI 과대 표시 → 신뢰 하락 | 높음 | 보수적 계산 기준 + 계산 근거 투명 공개 |
| 인사이트 인과관계 오류 | 중간 | "추정"임을 명시 + confidence 표시 + 사장 피드백 |
| LangGraph ↔ Next.js 복잡도 | 중간 | S1~S2는 AI SDK로, S3부터 LangGraph 전환 |

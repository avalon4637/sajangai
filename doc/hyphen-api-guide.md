# 하이픈(Hyphen) API 연동 가이드

> 사장 AI에서 하이픈 API를 통해 어떤 데이터를 수집하는지, 어떤 API를 연동해야 하는지 정리한 문서입니다.
> 작성일: 2026-03-20

---

## 1. 사장 AI가 하이픈에서 수집하는 데이터

| 데이터 종류 | 하이픈 API | 사장 AI 저장 위치 | 사용 에이전트 |
|-----------|-----------|-----------------|-------------|
| **배달앱 매출** | 배달의민족/쿠팡이츠/요기요 주문 내역 | `revenues` 테이블 | 세리 (매출 분석) |
| **배달앱 리뷰** | 배달앱 고객 리뷰 | `delivery_reviews` 테이블 | 답장이 (리뷰 분석) |
| **카드 매출** | 여신금융협회 카드 승인 내역 | `revenues` 테이블 | 세리 (매출 분석) |
| **카드 정산** | 카드사별 정산(입금) 내역 | 현금흐름 예측에 활용 | 세리 (현금흐름) |

## 2. 하이픈 가입 후 연동해야 할 API (우선순위별)

### 2-1. [필수] 배달앱 매출/리뷰 API

사장님이 배달앱을 사용 중이라면 가장 먼저 연동해야 합니다.

**연동 대상 플랫폼:**
- 배달의민족 (BAEMIN)
- 쿠팡이츠 (COUPANGEATS)
- 요기요 (YOGIYO)

**수집 데이터:**
| 항목 | 필드 | 설명 |
|------|------|------|
| 주문번호 | orderId | 고유 주문 식별자 |
| 주문일시 | orderDate, orderDatetime | 주문 발생 시간 |
| 정산일 | settlementDate | 실제 입금 예정일 |
| 주문금액 | orderAmount | 총 주문액 |
| 배달비 | deliveryFee | 배달비 |
| 수수료 | commissionFee, commissionRate | 플랫폼 수수료 |
| 정산금액 | netAmount | 실제 입금액 |
| 메뉴명 | menuName | 주문 메뉴 |
| 리뷰 별점 | rating (1-5) | 고객 평점 |
| 리뷰 내용 | content | 리뷰 텍스트 |
| 고객명 | customerName | 리뷰 작성자 |

**하이픈 콘솔에서 설정할 것:**
1. 배달앱 계정 연동 (각 플랫폼별 아이디/비밀번호 등록)
2. API 권한: 주문 내역 조회, 리뷰 조회 활성화
3. 수집 주기 설정 (사장 AI 기본: 유료 하루 5회 / 무료 체험 주 1회)

### 2-2. [필수] 카드매출 API (여신금융협회)

카드 결제를 받는 매장이라면 반드시 연동합니다.

**수집 데이터:**
| 항목 | 필드 | 설명 |
|------|------|------|
| 승인번호 | approvalNo | 카드 승인 식별자 |
| 승인일시 | approvalDate, approvalDatetime | 결제 시간 |
| 카드사 | cardCompany | 신한/KB/현대/롯데 등 |
| 카드번호 (마스킹) | cardNo | ****-****-****-1234 |
| 승인금액 | approvalAmount | 결제 금액 |
| 할부 | installments | 할부 개월 |
| 취소 여부 | isCancelled | 취소 거래 식별 |

**카드 정산 데이터:**
| 항목 | 필드 | 설명 |
|------|------|------|
| 정산일 | settlementDate | 카드사 입금일 |
| 카드사 | cardCompany | 정산 카드사 |
| 정산금액 | netAmount | 수수료 차감 후 입금액 |
| 수수료 | feeAmount | 카드사 수수료 |
| 거래건수 | transactionCount | 건수 |

**하이픈 콘솔에서 설정할 것:**
1. 여신금융협회 연동 (사업자번호 기반 자동 연결)
2. VAN사 정보 확인
3. 카드 정산 입금 계좌 확인

### 2-3. [선택] 홈택스 연동

세금 관련 데이터가 필요하면 연동합니다. (현재 사장 AI에서 미구현, 향후 확장)

- 매입/매출 세금계산서
- 현금영수증 내역
- 부가세 신고 데이터

## 3. 환경변수 설정

`.env.local`에 아래 변수를 설정합니다:

```env
# 하이픈 API 인증
HYPHEN_API_KEY=하이픈에서_발급받은_API_Key
HYPHEN_API_SECRET=하이픈에서_발급받은_API_Secret

# 테스트 모드 (실제 API 연동 전)
HYPHEN_TEST_MODE=true  # true면 mock 데이터 사용, false면 실제 API 호출
```

**API 키 발급 방법:**
1. [하이픈 콘솔](https://console.hyphen.im) 로그인
2. 프로젝트 생성 또는 기존 프로젝트 선택
3. API Key / Secret 발급
4. `.env.local`에 입력

---

## 4. 사장 AI 설정 화면에서의 연동

API 키 설정 후 사장 AI 웹에서:

1. **설정 > 데이터 연동** 페이지 이동 (`/settings/connections`)
2. 각 플랫폼별 "연결하기" 버튼 클릭
3. 하이픈 계정의 자격증명 입력 (AES-256-GCM으로 암호화 저장)
4. "동기화" 버튼으로 즉시 데이터 수집 테스트
5. 이후 자동 동기화 스케줄에 의해 주기적 수집

---

## 5. 동기화 구조

```
[하이픈 API]
    ↓ OAuth 토큰 인증 (oauth.ts)
    ↓
[sync-delivery.ts] → 배달앱 주문/리뷰 수집
[sync-card.ts]     → 카드 승인/정산 수집
    ↓ normalizer.ts (데이터 정규화)
    ↓
[sync-orchestrator.ts] → 전체 동기화 조율
    ↓
[Supabase] revenues, delivery_reviews 테이블 저장
    ↓
[세리/답장이 AI 엔진] → 분석 및 인사이트 생성
```

**자동 동기화 스케줄:**
- `/api/cron/sync` 엔드포인트
- 유료 플랜: 하루 5회 (약 5시간 간격)
- 무료 체험: 주 1회

## 6. 현재 구현 상태

| 기능 | 상태 | 비고 |
|------|------|------|
| 배달앱 매출 수집 | ✅ 구현 완료 | Mock 데이터 포함 |
| 배달앱 리뷰 수집 | ✅ 구현 완료 | Mock 데이터 포함 |
| 카드 승인 수집 | ✅ 구현 완료 | Mock 데이터 포함 |
| 카드 정산 수집 | ⚠️ 타입 정의만 | normalizer에 타입 있으나 sync 미구현 |
| OAuth 토큰 관리 | ✅ 구현 완료 | 캐싱 + 자동 갱신 |
| 자격증명 암호화 | ✅ 구현 완료 | AES-256-GCM |
| 동기화 오케스트레이터 | ✅ 구현 완료 | 에러 격리 포함 |
| 설정 UI | ✅ 구현 완료 | /settings/connections |
| 수동 동기화 API | ✅ 구현 완료 | POST /api/sync |
| 자동 스케줄러 | ✅ 구현 완료 | /api/cron/sync |

**`HYPHEN_TEST_MODE=true` (현재 상태):**
API 키 없이도 Mock 데이터로 전체 플로우를 테스트할 수 있습니다.
실제 API 키를 발급받은 후 `HYPHEN_TEST_MODE=false`로 변경하면 실제 데이터를 수집합니다.

## 7. 하이픈 가입 후 즉시 해야 할 것 (체크리스트)

- [ ] 하이픈 콘솔에서 API Key / Secret 발급
- [ ] `.env.local`에 `HYPHEN_API_KEY`, `HYPHEN_API_SECRET` 입력
- [ ] 배달앱 계정 연동 (배민/쿠팡/요기요 중 사용 중인 것)
- [ ] 카드매출 연동 (여신금융협회 - 사업자번호로 자동 연결)
- [ ] `HYPHEN_TEST_MODE=false`로 변경
- [ ] 사장 AI 설정 페이지에서 "동기화" 테스트
- [ ] 데이터 수집 확인 (대시보드에서 매출/리뷰 데이터 표시 확인)

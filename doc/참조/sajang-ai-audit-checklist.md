# sajang.ai AI 역량 점검 & 개선 체크리스트

> **목적**: 각 모듈이 AI의 강점을 제대로 활용하고 있는지 점검하고, "자동화(Level 1) → 해석(Level 2) → 판단/제안(Level 3)" 수준으로 끌어올릴 구체적 개선점을 도출한다.
>
> **사용법**: Claude Code에서 이 문서를 참조하며 각 모듈의 코드를 점검. 각 항목별로 [현재 구현 상태]를 확인하고, [개선 방향]에 따른 구체적 코드 변경 포인트를 찾는다.

---

## 1. 점장 — 홈·종합 브리핑 (Dashboard/Briefing)

### 현재 문제 진단
- 사장님이 로그인해도 "뭘 봐야 하는지" 모름 (Passive 모드)
- 데이터 요약만 존재, "왜?"와 "그래서 어떻게?" 가 없음
- 매일 접속할 이유를 만들지 못함

### 점검 항목

#### 1-1. 능동적 브리핑 카드 시스템
- [ ] 로그인 시 "오늘의 브리핑" 카드가 자동 생성되는가?
- [ ] 브리핑에 **어제/최근 7일 데이터 기반 변화 감지**가 포함되는가?
- [ ] 변화의 **원인 추정** (요일, 날씨, 시즌, 이벤트 등 컨텍스트)이 포함되는가?
- [ ] 구체적 **액션 제안** ("배달앱 쿠폰 등록 추천", "메뉴 X 재고 확인 필요")이 있는가?
- [ ] 브리핑의 우선순위 정렬이 되는가? (긴급도 × 임팩트 순)

**코드 점검 포인트:**
```
- src/app/(dashboard)/ 내 메인 페이지 컴포넌트 확인
- 브리핑 생성 로직 확인: AI API를 호출하는가, 아니면 정적 템플릿인가?
- 프롬프트에 "원인 분석"과 "액션 제안"이 포함되어 있는가?
- 데이터 fetch 시점: 로그인 시 자동 vs 사용자 클릭 필요?
```

#### 1-2. 핵심 알림 시스템
- [ ] 위험 신호 자동 감지 로직이 있는가? (매출 급락, 비용 급등, 현금 고갈 예측)
- [ ] 알림의 심각도 구분이 되는가? (🔴 긴급 / 🟡 주의 / 🟢 참고)
- [ ] 알림에서 바로 상세 분석 또는 시뮬레이션으로 이동할 수 있는가?
- [ ] 이전 알림 이력을 볼 수 있는가? (지난주 경고 → 이번 주 개선 여부 추적)

**코드 점검 포인트:**
```
- 알림/경고 관련 컴포넌트 존재 여부
- KPI 계산 엔진(src/lib/kpi/)에서 임계값 기반 경고 트리거 로직 확인
- 알림이 AI 생성인가 vs 규칙 기반인가?
```

#### 1-3. 일별 변화 내러티브
- [ ] "어제 vs 오늘", "이번 주 vs 지난주" 비교가 자연어로 생성되는가?
- [ ] 단순 숫자 나열이 아니라 **스토리텔링** 형태인가?
  - 나쁜 예: "매출 120만원, 전일 대비 -8%"
  - 좋은 예: "어제 매출이 전일 대비 8% 줄었어요. 화요일 평균과 비교하면 정상 범위이지만, 3주 연속 화요일 하락세가 보여요. 화요일 점심 세트 구성을 바꿔보는 건 어떨까요?"

**코드 점검 포인트:**
```
- AI 프롬프트 내 출력 형식 지시 확인
- 비교 기준 데이터(전일/전주/동일요일 평균)를 프롬프트에 함께 전달하는가?
- 시계열 데이터를 충분히 넘기고 있는가? (최소 4주치)
```

## 2. 세리 — 매출 분석 (Revenue Analysis)

### 현재 문제 진단
- 통계적 수치만 표시 (합계, 평균, 증감률)
- "그래서 뭘 해야 하는데?"라는 질문에 답을 못 함
- 엑셀 피벗 테이블과 차별점이 약함

### 점검 항목

#### 2-1. 패턴 감지 & 이상 탐지
- [ ] 요일별/시간대별/메뉴별 매출 패턴을 자동 분석하는가?
- [ ] 평소와 다른 이상치(anomaly)를 감지하여 표시하는가?
- [ ] 매출 트렌드 방향성을 예측하는가? (단순 전월비가 아닌 추세선)
- [ ] 계절성/주기성을 인식하는가? (월초 vs 월말, 비오는 날 vs 맑은 날)

**코드 점검 포인트:**
```
- src/lib/kpi/ 내 분석 로직이 단순 집계인가 vs 패턴 인식이 있는가?
- 차트(Recharts) 데이터에 추세선/이상치 마킹이 포함되는가?
- AI에게 매출 데이터를 넘길 때 충분한 히스토리를 포함하는가?
```

#### 2-2. 원인-결과 연결
- [ ] 매출 변동의 원인을 추정하는 로직/프롬프트가 있는가?
- [ ] 리뷰 데이터, 지출 데이터, 외부 요인(날씨/공휴일)과 **교차 분석**하는가?
- [ ] "매출이 떨어진 이유"를 사장님이 이해할 수 있는 말로 설명하는가?

**코드 점검 포인트:**
```
- 매출 분석 시 다른 테이블(리뷰, 지출, 고정비) 데이터를 함께 조회하는가?
- AI 프롬프트에 매출 데이터만 들어가는가, 맥락 데이터도 함께 들어가는가?
- 외부 데이터(공휴일/날씨 API) 연동 여부
```

#### 2-3. 실행 가능한 제안 생성
- [ ] 분석 결과에 기반한 구체적 액션 아이템을 AI가 생성하는가?
- [ ] 제안의 예상 효과(시뮬레이션 연결)를 보여주는가?
- [ ] 제안을 "이 제안 적용" → 시뮬레이션 화면으로 바로 연결하는가?

**코드 점검 포인트:**
```
- src/lib/simulation/ 과 매출 분석 모듈 간 연결 확인
- AI 응답 내 액션 아이템 구조화(JSON) 여부
- 제안 → 시뮬레이션 파라미터 자동 매핑 로직 유무
```

## 3. 답장이 — 리뷰 관리 (Review Management)

### 현재 문제 진단
- 자동 답변은 "잡일 경감" 수준이지 "경쟁력 강화"가 아님
- 리뷰 데이터에서 경영 인사이트를 추출하지 못함
- 답변 품질이 정형화되어 있어 AI 감성이 느껴지지 않음

### 점검 항목

#### 3-1. 리뷰 인사이트 추출
- [ ] 리뷰에서 반복 키워드/토픽을 자동 추출하는가?
- [ ] 긍정/부정 키워드를 분류하고 트렌드를 보여주는가?
  - 예: "'양'이 3개월 연속 부정 키워드 1위 → 포션 사이즈 검토 필요"
- [ ] 경쟁 매장 대비 강점/약점을 리뷰 기반으로 분석하는가?
- [ ] 리뷰 트렌드 변화를 주간/월간 리포트로 생성하는가?

**코드 점검 포인트:**
```
- 리뷰 데이터 저장 시 키워드 추출/분류 로직 유무
- AI 프롬프트에 리뷰 전체를 넘기는가 vs 최근 N건만 넘기는가?
- 리뷰 분석 결과를 별도 테이블에 저장하여 시계열 추적이 가능한가?
```

#### 3-2. 답변 품질 & 개인화
- [ ] 답변이 매장의 톤앤매너를 반영하는가? (사장님의 말투 학습)
- [ ] 리뷰 내용에 맞춘 구체적 답변인가 vs 범용 템플릿인가?
- [ ] 부정 리뷰에 대한 답변이 방어적이지 않고 건설적인가?
- [ ] 사장님이 답변을 수정한 이력을 학습하여 점점 개선되는가?

**코드 점검 포인트:**
```
- 리뷰 답변 프롬프트에 매장 정보, 이전 답변 스타일이 포함되는가?
- 사장님의 답변 수정 이력을 저장하고 프롬프트에 반영하는가?
- 답변 프롬프트의 구체성 수준 확인 (리뷰 원문, 별점, 메뉴, 방문 횟수 등)
```

#### 3-3. 리뷰 → 경영 연결
- [ ] "리뷰에서 나온 불만 → 메뉴/운영 개선 제안" 파이프라인이 있는가?
- [ ] 리뷰 기반 마케팅 포인트 추출이 되는가?
  - 예: "'분위기 좋다' 언급 42회 → 인스타 마케팅에 '분위기' 강조 추천"
- [ ] 리뷰 데이터가 세리(매출분석)와 연동되는가?
  - 예: "부정 리뷰 급증 시기와 매출 하락 시기 매칭"

**코드 점검 포인트:**
```
- 리뷰 모듈과 매출 분석 모듈 간 데이터 교차 참조 로직
- 리뷰 인사이트를 대시보드 브리핑에 포함하는 로직
- 마케팅 모듈(바이럴)과 리뷰 모듈 간 연결
```

## 4. 바이럴 — 마케팅 (Viral Marketing)

### 점검 항목

#### 4-1. 데이터 기반 마케팅 생성
- [ ] 매출/리뷰 데이터를 기반으로 마케팅 콘텐츠를 생성하는가?
- [ ] 매장의 강점을 데이터에서 자동 추출하여 마케팅 포인트로 사용하는가?
- [ ] 시즌/트렌드에 맞는 마케팅 제안이 자동으로 나오는가?
- [ ] 생성된 콘텐츠가 채널별(인스타, 네이버, 카카오)로 최적화되는가?

**코드 점검 포인트:**
```
- 마케팅 콘텐츠 생성 시 매출/리뷰 데이터를 AI에 함께 전달하는가?
- 채널별 포맷(글자 수, 해시태그, 이미지 비율) 가이드가 프롬프트에 있는가?
- 마케팅 효과 추적 (게시 후 매출 변화) 로직 유무
```

## 5. AI 챗 (AI Chat)

### 점검 항목

#### 5-1. 맥락 인식 대화
- [ ] 채팅 시 해당 매장의 전체 데이터(매출, 리뷰, 비용)를 맥락으로 제공하는가?
- [ ] "매출이 왜 떨어졌어?"라고 물으면 실제 데이터 기반으로 답하는가?
- [ ] 이전 대화 이력을 기억하여 연속 질문이 가능한가?
- [ ] 사장님의 질문 수준에 맞게 설명 난이도를 조절하는가?

**코드 점검 포인트:**
```
- src/lib/ai/ 내 채팅 프롬프트에 어떤 데이터가 system prompt로 들어가는가?
- tool use/function calling을 활용하여 실시간 데이터 조회가 가능한가?
- 대화 이력 관리 방식 (세션 기반 vs DB 저장)
```

#### 5-2. 선제적 질문 제안
- [ ] 빈 채팅 화면에서 "이런 거 물어보세요" 추천이 데이터 기반인가?
  - 나쁜 예: "매출을 분석해줘" (범용)
  - 좋은 예: "지난주 화요일 매출이 왜 낮았는지 분석해줘" (맥락 반영)
- [ ] 빠른 질문 버튼이 현재 매장 상황에 맞게 동적으로 바뀌는가?

**코드 점검 포인트:**
```
- 빠른 질문/추천 질문 리스트가 하드코딩인가 vs AI 동적 생성인가?
- 추천 질문 생성 시 최근 데이터 변화를 반영하는가?
```

## 6. 시뮬레이션 (What-if Simulation)

### 점검 항목

#### 6-1. AI 기반 시나리오 추천
- [ ] 사장님이 직접 시나리오를 설정해야만 하는가? (Passive)
- [ ] AI가 현재 데이터 기반으로 "해볼 만한 시나리오"를 추천하는가? (Active)
  - 예: "현재 인건비 비율이 높습니다. 직원 1명 감축 시 월 순이익 +42만원 예상 → 시뮬레이션 보기"
- [ ] 시뮬레이션 결과에 대한 AI 해석이 붙는가?
  - 나쁜 예: "순이익 340만원 → 382만원"
  - 좋은 예: "순이익이 12% 개선되지만, 피크 시간대 서비스 품질 저하 리스크가 있습니다"

**코드 점검 포인트:**
```
- src/lib/simulation/ 내 시나리오 추천 로직 유무
- 시뮬레이션 결과를 AI 프롬프트에 넘겨 해석을 생성하는가?
- 시뮬레이션 파라미터가 사장님에게 직관적인가? (슬라이더 등)
```

## 7. 크로스 모듈 통합 점검

### 현재 가장 큰 구조적 문제
각 모듈이 **사일로(Silo)**로 동작하여 AI의 최대 강점인 **교차 분석**을 못 살림.

#### 7-1. 데이터 교차 분석
- [ ] 매출 분석 시 리뷰 데이터를 함께 참조하는가?
- [ ] 리뷰 분석 시 해당 시기 매출 데이터를 교차하는가?
- [ ] 비용/고정비 데이터가 매출 분석의 수익성 계산에 반영되는가?
- [ ] 마케팅 제안 시 매출/리뷰 트렌드를 근거로 활용하는가?

**코드 점검 포인트:**
```
- 각 모듈의 AI 프롬프트가 해당 모듈 데이터만 사용하는가?
  → 가장 흔한 문제. 매출 분석 프롬프트에 매출만, 리뷰 프롬프트에 리뷰만.
- 통합 데이터 fetch 함수가 있는가? (여러 테이블 조인)
- AI API 호출 시 context window 활용도 (데이터를 충분히 넣고 있는가?)
```

#### 7-2. 통합 인사이트 생성
- [ ] "이번 주 종합 리포트"를 전 모듈 데이터 기반으로 생성하는가?
- [ ] 모듈 간 인과관계를 AI가 연결해주는가?
  - 예: "리뷰에서 '느리다' 불만 ↑ → 같은 시기 배달 매출 ↓ → 배달 프로세스 점검 필요"

## 8. AI 프롬프트 품질 점검

### 가장 빠르게 제품 가치를 올릴 수 있는 영역

#### 8-1. 프롬프트 구조 점검
- [ ] 각 모듈의 AI 프롬프트에 **역할 정의**가 명확한가?
  - "당신은 F&B 소상공인 경영 컨설턴트입니다" vs "데이터를 분석해주세요"
- [ ] 프롬프트에 **출력 형식**이 구조화되어 있는가? (JSON, 섹션별 구분)
- [ ] **Few-shot 예시**가 포함되어 있는가?
- [ ] 프롬프트에 **매장 컨텍스트**(업종, 규모, 위치, 영업시간)가 포함되는가?
- [ ] 프롬프트가 **한국어 소상공인**에 맞게 최적화되어 있는가?

**코드 점검 포인트:**
```
- src/lib/ai/ 내 모든 프롬프트 파일 리스트업
- 각 프롬프트의 system prompt 내용 전문 확인
- 프롬프트에 전달되는 데이터의 양과 구조 확인
- temperature, max_tokens 등 파라미터 설정 확인
```

#### 8-2. 데이터 → 프롬프트 전달 점검
- [ ] 프롬프트에 전달하는 데이터가 **충분한가**? (최소 4주 데이터)
- [ ] 데이터 포맷이 AI가 이해하기 좋게 **구조화**되어 있는가?
- [ ] 불필요한 데이터가 토큰을 낭비하고 있지 않은가?
- [ ] 데이터 전달 시 **요약 통계 + 원본 샘플** 조합을 사용하는가?

## Claude Code 사용 가이드

### 1단계: 현황 파악
```
각 모듈 폴더를 순회하며 AI 관련 코드를 찾아 현재 구현 수준을 파악해주세요:
- src/app/(dashboard)/ 내 각 페이지의 데이터 fetch 및 AI 호출 로직
- src/lib/ai/ 내 모든 프롬프트 및 AI 관련 유틸리티
- src/lib/kpi/ 내 분석 로직
- src/app/api/ai/ 내 API 라우트
```

### 2단계: 프롬프트 수집
```
프로젝트 전체에서 AI 프롬프트(system prompt, user prompt)를 모두 수집하여
각 프롬프트가 위 체크리스트의 어떤 항목에 해당하는지 매핑해주세요.
```

### 3단계: 개선 우선순위 실행
```
위 우선순위 표의 1~3순위부터 구체적 코드 변경 사항을 제안해주세요.
특히:
- 프롬프트 개선안 (before/after)
- 데이터 fetch 확장 (어떤 테이블을 추가로 조회해야 하는가)
- 컴포넌트 변경 (능동적 브리핑 카드 추가 등)
```

## 핵심 원칙 요약

> **AI의 가치 = "사장님이 혼자서는 절대 못 하는 것"을 해주는 것**
>
> - Level 1 (자동화): 엑셀로도 되는 일 → ❌ 차별화 없음
> - Level 2 (해석): 패턴을 발견하고 원인을 추정 → ⚠️ 있으면 좋지만 부족
> - Level 3 (판단/제안): 구체적 행동을 제안하고 결과를 예측 → ✅ 이것이 PMF의 핵심
>
> 모든 모듈에서 **"그래서 뭘 하라고?"** 에 답할 수 있어야 한다.
